HTML

Ideas principales.

  • Tim Berners-Lee desarrollo el HTML
  • El HTML es el componente más básico de la web.
  • El Hypertext se puede leer en cualquier orden.
  • Se utilizan los caracteres de menor y mayor para etiquetar texto, imagenes y mas contenido.
  • La mayoria de las etiquetas tienen inicio y fin.
  • Para cerrar una etiquta antes de escribir el nombre debe llevar una diagonal /.
  • Entre la etiquetas debe estar el contenido.

Apuntes de la clase.

Desarrollador del HTML: Tim Berners-Lee.
Hypertext. Se puede leer en el orden qué sea.
HTML. (HyperText Markup Language) es el componente más básico de la web. Define el significado y la estructura del contenido web.

  • Utiliza la antilambda o los diples (ósea los caracteres menores qué y mayor que) para etiquetar texto, imágenes y otro contenido para mostrarlo en un navegador web.
  • La gran mayoría de las etiquetas deben tener un inicio y un fin.
  • El inicio de una etiqueta es normal, en otras palabras, lleva los dos caracteres y dentro de estos se encuentra el nombre.
  • Para el cierre de la etiqueta antes de escribir el nombre de la etiqueta debemos escribir una diagonal /.
  • Entre el inicio y el fin de la etiqueta debe de estar el contenido.
Estructura básica de un archivo HTML.
  1. !DOCTYPE html Declaracion del HTML5.
  2. html Inicio del archivo HTML.
  3. head Apertura etiqueta head.
  4. /head Cierre de la etiqueta head.
  5. body Apertura de etiqueta body.
  6. /body Cierra de etiqueta body.
  7. /HTML Cierre del archivo HTML.

Resumen.

El HTML (HyperText Markup Language) es el componente mas basico de la web, el cual fue desarrollado por Tim Berners-Lee. EL HyperText se puede leer en el orden que sea. Se utilizan los diples (mejor conocidos como caracteres de menor que y mayor que) para etiquetar el texto, las imagenes y mas contenido, esto para poder ser mostrado en la web. La mayoria de las eqtiquetas deben tener un inicio y un fin y dentro de estas debe de estar el contenido.

Texto en HTML

Ideas principales.

  • En HTML hay seis diferentes encabezados.
  • Se deben usar los encabezados en el orden correcto de la jerarquía.
  • No se deben usar más de tres encabezados por páginas.
  • Para agregar un parrafo en HTML se debe agregar un elemento p.
  • En HTML existen dos tipos de listas las ordenadas y las no ordenadas.
  • En las ordendas importa el orden de los elementos.
  • En las desordenadas no importa el orden de la información.

Apuntes de la clase.

Texto en HTML. La mayor parte del texto estructurado está compuesto por encabezados, párrafos, independientemente si lees una historia, un periódico, un libro de texto una revista. Encabezados. En HTML hay seis diferentes encabezados.
Encabezados. En HTML hay seis diferentes encabezados.
Buenas prácticas.

  • Es mejor usar solo un h1 por página.
  • Asegúrate de que usas los encabezados en el orden correcto de la jerarquía. Procura no usar más de tres niveles de encabezado por página.
Párrafos. En HTML, cada párrafo debe de estar delimitado por un elemento p.
Listas. Hay dos diferentes tipos de listas.
  • Listas no ordenadas. Se usan para marcar listas de artículos cuyo orden no es importante.
  • Cada lista desordenada comienza con un ul (la ul solo puede tener li)
Listas ordenadas. Son aquellas donde el orden de los elementos si importa.

Resumen.

Para agregar texto en HTML se utiliza la misma estructura de un libro, de un texto o de una revista. En HTML existen 6 tipos de encabezados, estos encabezados se deben usar en el orden correcto de la jerarquia, se debe preocurar no usar más de tres niveles de encabezado por página. Los parrafos se deben agregar en un elemento p. También existen dos tipos de listas, las ordenadas y las no ordenadas.

Imagenes en HTML.

Ideas principales.

  • Las imágenes en la web ayudan a los buscadores a indexar las páginas.
  • Proporcionan un descanso en la lectura del usuario.
  • Sirven de explicación gráfica para la inofrmación que estas aportando.
  • Se pueden agregar imágenes desde una carpeta o desde la web.
  • El atributo width determina el ancho y el atributo height determina la altura de una imagen.
  • El atributo alt es un texto que se muestra en lugar de una imagen cuando esta no esta disponible.
  • El atributo alt es importante porque es la información que puede extraer el buscador.
  • Si se reduce una imagen en HTML no se reduce su peso.

Además, proporcionan un descanso en la lectura del usuario y muchas veces sirven de explicación gráfica para lo que está contando.

  • En diferentes folders.Si no se encuentra en la misma carpeta, la ruta se escribe de la siguiente manera. img src=”carpeta/archivo.jpg” o img src=”carpeta/carpeta/archivo.jpg”.

Enlaces en HTML.

Ideas principales.

  • Los links se puden usar para cualquier cosa, por ejemplo, una imagen, un texto, un boton, un video, entre otros.
  • Los links deben seguir una sintaxis.
  • Un enlace que no sea visitado aparece subrayado y en azul.
  • Un enlace que ya fue visitado estará subrayado y en morado.
  • Un enlace que esta activo estará subrayado y en rojo.
  • Se puede usar una imagen como link poniendo img dentro de la etiqueta a.
  • El atributo target sirve para especificar donde se debe abrir el documento vinculado.

CSS

Ideas principales.

  • El CSS define la apariencia del HTML.
  • Es un mecanismo simple para agregar estilo o diseño.
  • Se pueden agregar fuentes, colores, espaciado a los documentos web.
  • Existen tres maneras de poner el estilo de CSS en el HTML.
  • Hoja de estilo incrustada la cual se coloca en la cabecera del documento.
  • Hoja de estilo en línea la cual se puede aplicar en cualquier elemento del HTML.
  • Hoja de estilo externa la cual se puede enlazar al HTML mediante el elemento link.
  • El código CSS se compone de sus propias reglas.
  • El CSS tendrá un orden en cascada.
  • Un estilo en línea tiene la mayor prioridad y anulara los estilos externos e internos y los valores predeterminados.

Apuntes de la clase.

CSS Es un lenguaje que define la apariencia de un documento escrito en un lenguaje de marco. Son un mecanismo simple para agregar estilo o diseño, por ejemplo, fuentes, colores, espaciado a documentos web. A los elementos de la página web creados con HTML se les dará la apariencia que se desee utilizando CSS: formas, tamaños, colores, márgenes, tipos de letras, etc.
Como agregar CSS al HTML. Hay tres formas de poner el estilo de CSS en el HTML:

  1. Hoja de estilo incrustada. Puede incrustarse en un documento con la etiqueta style, se coloca en la cabecera head del documento. Puede usarse en un documento único. Si la misma hoja de estilo se usa para múltiples documentos, entonces sería más apropiado una hoja de estilo extra. Nivel 2.
  2. Hoja de estilo en línea. Puede usarse mediante el atributo style. Puede ser aplicado a cualquier elemento HTML (incluyendo en body). Pierde muchas de las ventajas de las hojas de estilo al mezclar el contenido con presentación. Si el estilo debe aplicarse a un único elemento, pero con solo cierto medio, use el atributo ID en lugar del atributo STRYLE. Nivel 1.
  3. Hoja de estilo externa. Puede ser enlazada a un documento HTML mediante el elemento link de HTML. La etiqueta link se coloca en la cabecera head del documento. El atributo type es opcional se usa para especificar un tipo de medio-text/css. Nivel 3.
Sintaxis de una regla CSS. Todo el código CSS se compone de reglas. Una regla es el conjunto de propiedades que se le van a aplicar a un elemento HTML. #principal{Font-family:arial;}
Orden en cascada. Todos los estilos de una página se “encadenaran” en una hoja de estilo “virtual” según las siguientes reglas, donde el numero uno tiene la máxima prioridad:
  1. Estilo en línea (dentro de un documento HTML).
  2. Hojas de estilo externas e internas (en la sección del encabezado).
  3. Navegador predeterminado.

Resumen.

El CSS es el que define la apariencia de un documento, es un mecanismo simple para agregar estilo o diseño, sirve para agregar fuentes, colores o espaciados a los documentos web. A los elementos de la página web creados en HTML se les dará la apariencia utilizando CSS. Existen tres maneras de poner el estilo CSS en ele HTML, hoja de estilo incrustada, hoja de estilo en línea y la hoja de estilo externa. El CSS tiene su propia sintaxis y sus propias reglas, el CSS tiene un orden en cascada.

Proceso de diseño.

Ideas principales.

  • El diseño necesita una metodología, técnicas y procedimientos ideados para asegurar que un sitio tendrá los niveles de usabilidad requeridos.
  • Asume todo el proceso de diseño y desarrollo del sitio web debe estar conducido por el usuario, sus necesidades, características, objetos.
  • Se debe centrar el diseño en sus usuarios.
  • Se debe involucrar desde el comienzo a los usuarios en el proceso de desarrollo del sitio.
  • El proceso de Diseño Web Centrado en el Usuario se divide en varias fases o etapas.
  • Las fases de diseño, prototipado y evaluación son cíclicas e iterativas.
  • En la planificación se identifica los objetivos del sitio, así como las necesidades, requerimientos y objetivos de la audiencia.

Apuntes de la clase.

Diseño web centrado en el usuario. Para asegurar que un sitio con los niveles de usabilidad requeridos, el diseño necesita de una metodología, de técnicas y procedimientos ideados para tal fin. Se caracteriza por asumir que todo el proceso de diseño y desarrollo del sitio web debe estar conducido por el usuario, sus necesidades, características, objetos.
Centrar el diseño en sus usuarios (en lugar de centrarlo en las posibilidades tecnológicas o en nosotros mismos como diseñadores) implica:

  1. Involucrar desde el comienzo a los usuarios en el proceso de desarrollo del sitio; conocer cómo son, qué necesitan, para que usan el sitio.
  2. Proveer el sitio con los propios usuarios; investigar cómo ante el diseño, como es su experiencia de uso;
  3. Innovar siempre con el objetivo claro de mejorar la experiencia del usuario.
  4. El proceso de Diseño Web Centrado en el Usuario se divide en varias fases o etapas, algunas de las cuales tienen carácter iterativo.
  5. Las fases de “diseño”, “prototipado” y “evaluación” son cíclicas e iterativas.
Planificación.
  1. En esta etapa se identifica los objetivos del sitio, así como las necesidades, requerimientos y objetivos de la audiencia potencial.
  2. Se trata de establecer un equilibrio entre lo que cede ofertar el proveedor y lo que necesita el usuario.
  3. El sitio web - sus contenidos y diseño - debe servir de medio pata lograr los objetivos por parte de proveedor y usuario.
  4. La respuesta a estas preguntas se resuelve estudiando a la audiencia a través de métodos de investigación. Éstos engloban métodos de aproximación contextual, estudios de campo o etnográficos, métodos de aproximación por grupos y métodos de aproximación individual (encuestas, cuestionarios y entrevistas).
  5. Cuanto más conozcamos a la audiencia, más adaptados será el diseño y más satisfactoria la experiencia del usuario final.
  6. La etapa de planificación se basa casi completamente en recoger, analizar y ordenar toda la información posible, con el objetivo de tener una base sólida sobre la que poder tomar decisiones de diseño en las siguientes etapas del proceso.

Resumen.

El diseño web centrado en el usuario es importante para asegurar que un sitio tenga los niveles de usabilidad requeridos. Se caracteriza por asumir todo el proceso de diseño y el desarrollo del sitio web. Se debe centrar el diseño a los usuarios, desde un comienzo se debe involucrar a los usuarios. En la etapa de planificación se deben identificar los objetivos del sitio y las necesidades de los usuarios. Se debe estudiar a la audiencia a través de métodos de investigación.
